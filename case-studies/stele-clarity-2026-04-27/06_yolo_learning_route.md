# YOLO 是什么，我应该怎么学

YOLO 是一种目标检测方法。它做的事可以简单理解成：给一张图，找出图里有哪些对象，并画出它们的位置框。

比如一张汉画像石图片，YOLO 可以尝试找：

- 人
- 马
- 车
- 乐器
- 建筑
- 动物
- 日月
- 榜题文字区

它输出的不是“这幅图的意义”，而是：

```text
类别 + 位置框 + 置信度
```

例如：

```text
马，位置框 [x1, y1, x2, y2]，置信度 0.82
人，位置框 [x1, y1, x2, y2]，置信度 0.76
```

## YOLO 在汉画像石里能帮什么忙

它比较适合做第一步：把图像里的对象先找出来。

可以用在：

- 批量找人物。
- 批量找马和车。
- 找榜题文字区。
- 找乐舞场景里的乐器和舞者。
- 作为人工标注的预处理。

但它不能直接替代图像学判断。比如“这是不是伏羲女娲”“这是不是升仙图像”“这是不是某个历史故事”，这些还是要结合图像、文献和专家判断。

## 我需要先懂哪些概念

先懂这些就够了：

- 分类：判断整张图是什么。
- 检测：找出图里对象的位置。
- 分割：把对象轮廓抠出来。
- 标注：人工给图片画框、写标签。
- 训练集：给模型学习的图片。
- 验证集：训练过程中检查效果的图片。
- 测试集：最后检验模型的图片。
- 精确率：模型找出来的东西有多少是真的。
- 召回率：真实存在的东西有多少被模型找到了。

暂时不用先学：

- 神经网络公式。
- 反向传播。
- 损失函数推导。
- PyTorch 底层代码。

这些以后需要时再补。

## 第一阶段：先学会标注，1-2 周

先不要急着训练模型。没有好标注，模型学不到东西。

练习：

1. 选 30 张汉画像石图片。
2. 只标三类：人、马、车。
3. 用 VIA、Label Studio 或 CVAT 画框。
4. 每张图标完后写一句备注：哪里不好判断。

这一阶段要形成标注规则。比如：

- 残缺半个人算不算“人”？
- 人和神人先不先分开？
- 车轮单独标，还是只标整辆车？
- 线描、拓片、照片能不能混在一起训练？

这些问题比模型本身更重要。

## 第二阶段：跑一个小模型，1-2 周

工具可以先用 Ultralytics YOLO。它已经把训练流程包装好了，不需要自己从零写代码。

最小目标：

- 30-100 张图。
- 3 个类别：人、马、车。
- 训练一次。
- 看新图上能不能大致找出来。

这一步不要追求高指标，只要跑通流程。

## 第三阶段：看错误，2 周

模型第一次肯定会错。重点是把错误分清楚。

常见错误：

- 人太小，漏检。
- 线条太浅，漏检。
- 装饰纹被错认为人。
- 马和其他动物混淆。
- 类别定义太细，模型学乱。
- 标注者标准不一致。

做一张错误表：

```text
图片编号：
模型错在哪里：
可能原因：
要不要改类别：
要不要补标注：
```

## 第四阶段：扩数据，1-2 个月

等小流程跑通，再扩到 200-500 张图。

第一批类别建议不要超过 10 个：

- 人
- 马
- 车
- 建筑
- 乐器
- 鸟
- 兽
- 日
- 月
- 榜题文字区

先大类，后细分。比如先标“人”，等数据够了，再分“舞者、乐师、骑者、御者、神人”。

## 第五阶段：接到结构化数据里

YOLO 的结果最后要进入结构化表，而不是只停在一张带框图片。

可以这样存：

```json
{
  "image_id": "hhs_001",
  "objects": [
    {
      "class": "马",
      "bbox": [120, 80, 260, 150],
      "source": "yolo",
      "confidence": 0.82,
      "human_status": "confirmed"
    }
  ]
}
```

还要有人工作业字段：

- `confirmed`：确认。
- `rejected`：否定。
- `uncertain`：不确定。
- `corrected`：人工改过。

## 一个适合起步的小题目

题目可以先定成：

```text
汉画像石车马出行图像的目标检测和结构化试验
```

范围：

- 100 张车马出行相关图像。

先标：

- 人
- 马
- 车
- 车轮
- 御者
- 骑者
- 建筑
- 榜题文字区

最后做：

- YOLO 检测。
- 人工校正。
- 对象关系整理，比如“人骑马”“马拉车”“人在车中”。
- 一个简单检索表。

这个题目比较现实，因为车马出行对象相对明确，也容易和汉代礼制、交通、墓葬图像研究联系起来。

## 我不应该一开始做什么

- 不要一开始做全部汉画像石自动理解。
- 不要一开始标几十上百类。
- 不要把 AI 输出直接当结论。
- 不要只看模型准确率，要看它到底错在哪里。
- 不要忽略人工确认字段。

## 学习顺序

```text
知道 YOLO 是目标检测
  -> 学会人工标注
  -> 标 30-100 张小数据
  -> 跑一次 Ultralytics YOLO
  -> 看错误
  -> 改标注规范
  -> 扩到 200-500 张
  -> 接入 JSON/CSV 结构化数据
```

## 英文资料怎么读

YOLO 的英文资料很多，先不用精读论文公式。

先看：

- Ultralytics 文档里的 install、train、predict。
- YOLO 数据格式说明。
- 标注工具如何导出 YOLO 格式。
- 汉画像石 YOLO-v5 那篇论文的摘要、数据集、类别、实验结果。

每看一份资料，只写中文笔记：

```text
这个工具能做什么：
我需要准备什么数据：
命令怎么跑：
输出是什么：
哪里还没看懂：
```
