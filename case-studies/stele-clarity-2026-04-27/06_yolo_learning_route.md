# YOLO 是什么，以及文科背景如何学习

整理日期：2026-04-28。

## 1. YOLO 是什么

YOLO 是 **You Only Look Once** 的缩写，意思是“只看一遍”。它是一类实时目标检测模型。

普通图像分类回答的是：

```text
这张图是什么？
```

比如：这是一张汉画像石。

YOLO 回答的是：

```text
图里有哪些东西？它们分别在哪里？
```

比如：

```text
人物：x=120, y=80, w=60, h=140
马：x=240, y=130, w=180, h=90
车：x=430, y=120, w=220, h=110
建鼓：x=700, y=100, w=80, h=150
榜题文字区：x=40, y=30, w=120, h=40
```

对汉画像石来说，YOLO 的作用不是“自动解释图像意义”，而是先帮你找出图像里的对象候选：人物、马、车、乐器、建筑、动物、神话人物、榜题区域等。

## 2. YOLO 能做什么，不能做什么

### 能做

- 在图像里找对象位置。
- 给对象分类。
- 批量处理大量图片。
- 训练一个“汉画像石专用检测器”。
- 辅助标注，提高人工整理效率。

### 暂时不能指望它直接做

- 判断整幅图像的深层图像学意义。
- 自动识别复杂故事题材。
- 区分所有神话人物和历史人物。
- 替代专家释读。
- 在没有标注数据时凭空学会汉画像石。

更准确地说，YOLO 是“结构化流水线里的第一步工具”，不是最终解释者。

## 3. 对汉画像石的典型用途

### 第一层：对象检测

先训练它识别：

- 人
- 马
- 车
- 建筑
- 乐器
- 鸟兽
- 日月
- 榜题文字区

### 第二层：专门对象检测

数据多以后再训练：

- 伏羲
- 女娲
- 西王母
- 东王公
- 羽人
- 乐舞人物
- 骑者
- 御者

### 第三层：题材辅助判断

YOLO 检测结果可以作为题材判断依据：

```text
检测到：车 + 马 + 御者 + 随从
可能题材：车马出行
```

```text
检测到：人首蛇身男女 + 日月 + 规矩
可能题材：伏羲女娲
```

但这个“可能题材”应由规则、知识图谱或大模型结合文献再判断，最后仍需人工确认。

## 4. 学 YOLO 需要什么基础

### 必须学

- 图像标注是什么。
- 训练集、验证集、测试集是什么。
- 框选 bbox、多边形 polygon、类别 label 是什么。
- 精确率 Precision、召回率 Recall、大致是什么意思。
- 过拟合是什么意思。
- 怎么判断模型有没有用。

### 可以后学

- 神经网络内部结构。
- 卷积、注意力机制、损失函数。
- 反向传播和优化器。
- PyTorch 代码细节。

### 文科生最需要掌握的不是公式

你最重要的能力是：

- 定义类别体系。
- 判断什么算同一类。
- 制定标注规范。
- 审核模型识别错在哪里。
- 把检测结果转化为图像学和文献研究问题。

这正是文科背景的优势。

## 5. 最小学习路线

### 第 1 周：理解目标检测

目标：

- 分清分类、检测、分割。
- 知道 YOLO 输出的是“类别 + 位置框 + 置信度”。

学习内容：

- 看 Ultralytics YOLO object detection 文档。
- 看原始 YOLO 论文摘要即可，不必读公式。
- 找 10 张图片，手工画出“人、马、车”的框。

成果：

- 能解释“YOLO 对汉画像石有什么用”。

### 第 2 周：学习标注

目标：

- 会用 VIA、Label Studio 或 CVAT 做框选标注。
- 知道 YOLO 数据格式大概长什么样。

练习：

- 选 30 张汉画像石图片。
- 只标 3 类：人、马、车。
- 每张图最多先标主要对象，不追求完美。

成果：

- 一个小型标注数据集。

### 第 3 周：第一次训练

目标：

- 用现成 YOLO 工具训练一次。

推荐工具：

- Ultralytics YOLO。

练习：

- 用 30-100 张图训练一个玩具模型。
- 看模型能不能在新图上找出人、马、车。

成果：

- 第一个可运行的汉画像石对象检测模型。

### 第 4 周：理解评估和错误

目标：

- 不迷信模型结果。
- 会看哪些对象漏检，哪些对象误检。

练习：

- 把错误分成几类：残损、太小、背景复杂、类别定义不清、标注不一致。
- 修改标注规范。
- 再训练一次。

成果：

- 一份错误分析表。

### 第 2 个月：扩展数据和类别

目标：

- 从玩具模型变成可用原型。

建议：

- 图片扩到 200-500 张。
- 类别控制在 8-15 类。
- 优先标高频、边界相对明确的对象。

第一批建议类：

- 人
- 马
- 车
- 建筑
- 乐器
- 鸟
- 兽
- 榜题文字区
- 日
- 月

成果：

- 一个可以辅助批量标注的检测器。

### 第 3 个月：从检测到结构化

目标：

- 把 YOLO 输出接到汉画像石结构化数据里。

练习：

- 每个检测框生成 JSON。
- 人工确认或修改。
- 添加关系：人骑马、人驾车、马拉车、人物持器物。
- 添加题材：车马出行、乐舞百戏、伏羲女娲等。

成果：

- 一个“图像 -> 对象 -> 关系 -> 题材”的小型结构化样例。

## 6. 实操路线

### 最小项目：车马出行检测

范围：

- 100 张车马出行相关汉画像石。

类别：

- 人
- 马
- 车
- 车轮
- 御者
- 骑者
- 建筑
- 榜题文字区

步骤：

1. 收集图片。
2. 建立文件名和元数据表。
3. 用 Label Studio/CVAT 标注。
4. 导出 YOLO 格式。
5. 用 Ultralytics YOLO 训练。
6. 对新图预测。
7. 人工校正。
8. 保存结构化 JSON。

## 7. 你应该避免的坑

### 坑 1：类别太多

一开始不要标 100 类。类别越多，标注越难，模型越乱。

### 坑 2：类别定义不清

比如“人”“神人”“羽人”“仙人”如果边界不清，模型会学糊。可以先标大类“人物”，后续再细分。

### 坑 3：只看准确率

汉画像石里漏掉一个关键小人物，研究意义可能很大。不能只看总体数字，要看具体错误。

### 坑 4：用现代照片数据集思维套古代图像

汉画像石不是街景照片。线刻、拓片、残损、风格化造型都会让模型困难。

### 坑 5：AI 结果没有人工确认字段

结构化数据里必须区分：

- 模型预测
- 人工确认
- 人工否定
- 不确定

## 9. 一句话学习策略

你不必先学完整深度学习。最有效的路径是：

```text
懂目标检测概念
  -> 学会标注
  -> 训练一个小模型
  -> 看错误
  -> 改标注规范
  -> 扩数据
  -> 接入结构化数据库
```
