# YOLO 学习路线

YOLO 是目标检测方法，用于在图像中识别对象类别并给出位置框。对于汉画像石，YOLO 可以辅助检测人物、马、车、乐器、建筑、动物、日月和榜题文字区。

YOLO 输出内容包括：

```text
类别 + 位置框 + 置信度
```

示例：

```text
马，位置框 [x1, y1, x2, y2]，置信度 0.82
人，位置框 [x1, y1, x2, y2]，置信度 0.76
```

YOLO 适合做对象候选检测，不适合直接完成图像学解释。伏羲女娲、升仙图像、车马出行、历史故事等题材判断仍需结合图像、文献和人工审核。

## 适用任务

- 批量检测人物。
- 批量检测马和车。
- 检测榜题文字区。
- 检测乐舞场景中的乐器和舞者。
- 辅助人工标注。

## 基础概念

先掌握以下概念：

- 分类：判断整张图是什么。
- 检测：找出图中对象的位置。
- 分割：标出对象轮廓。
- 标注：人工给图片画框并写标签。
- 训练集：给模型学习的图片。
- 验证集：训练过程中检查效果的图片。
- 测试集：最终检验模型的图片。
- 精确率：模型检出的对象中有多少正确。
- 召回率：真实对象中有多少被模型检出。

暂缓学习内容：

- 神经网络公式。
- 反向传播。
- 损失函数推导。
- PyTorch 底层代码。

## 阶段 1：图像标注，1-2 周

训练前先建立标注规范。标注质量直接影响模型效果。

练习：

1. 选择 30 张汉画像石图片。
2. 只标三类：人、马、车。
3. 使用 VIA、Label Studio 或 CVAT 画框。
4. 每张图记录难以判断的位置。

需要提前规定的问题：

- 残缺半个人是否标为“人”。
- 人、神人、羽人是否先合并为大类。
- 车轮是否单独标注。
- 线描、拓片、照片是否混合训练。

阶段产出：

- 一个小型标注集。
- 一份标注规范草稿。

## 阶段 2：第一次训练，1-2 周

推荐使用 Ultralytics YOLO。该工具已封装训练流程，不需要从零编写神经网络。

最小训练目标：

- 30-100 张图。
- 3 个类别：人、马、车。
- 完成一次训练。
- 在新图上测试检测效果。

阶段目标是跑通流程，不追求高指标。

## 阶段 3：错误分析，2 周

第一次模型通常会出现漏检和误检。错误分析比指标更重要。

常见错误：

- 人物太小导致漏检。
- 线条太浅导致漏检。
- 装饰纹被误检为人物。
- 马和其他动物混淆。
- 类别定义过细导致模型混乱。
- 标注者标准不一致。

错误表字段：

```text
图片编号：
错误类型：
错误位置：
可能原因：
类别是否需要调整：
是否需要补充标注：
```

## 阶段 4：扩展数据，1-2 个月

流程跑通后，再扩展到 200-500 张图。

第一批类别控制在 10 个以内：

- 人
- 马
- 车
- 建筑
- 乐器
- 鸟
- 兽
- 日
- 月
- 榜题文字区

类别从大类开始。数据量增加后，再细分舞者、乐师、骑者、御者、神人等类别。

## 阶段 5：接入结构化数据

YOLO 结果需要进入结构化表，而不是只保存带框图片。

示例 JSON：

```json
{
  "image_id": "hhs_001",
  "objects": [
    {
      "class": "马",
      "bbox": [120, 80, 260, 150],
      "source": "yolo",
      "confidence": 0.82,
      "human_status": "confirmed"
    }
  ]
}
```

人工审核字段：

- `confirmed`：确认。
- `rejected`：否定。
- `uncertain`：不确定。
- `corrected`：人工修改。

## 起步题目

题目：

```text
汉画像石车马出行图像的目标检测和结构化试验
```

范围：

- 100 张车马出行相关图像。

初始类别：

- 人
- 马
- 车
- 车轮
- 御者
- 骑者
- 建筑
- 榜题文字区

工作内容：

- YOLO 检测。
- 人工校正。
- 对象关系整理，如“人骑马”“马拉车”“人在车中”。
- 建立简单检索表。

车马出行适合作为起步题材，因为对象相对明确，也容易与汉代礼制、交通和墓葬图像研究结合。

## 注意事项

- 初期不做全部汉画像石自动理解。
- 初期不设置过多类别。
- AI 输出只作为候选。
- 评估时查看具体错误，不只看总体准确率。
- 结构化数据中保留人工确认字段。

## 学习顺序

```text
目标检测概念
  -> 人工标注
  -> 30-100 张小数据
  -> Ultralytics YOLO 训练
  -> 错误分析
  -> 标注规范修订
  -> 200-500 张扩展数据
  -> JSON/CSV 结构化数据
```
