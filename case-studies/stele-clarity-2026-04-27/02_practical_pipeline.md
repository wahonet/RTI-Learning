# 针对这批碑刻照片的可执行方案

## 先给结论

如果目标只是“肉眼更好看”，可以先跑图像增强。  
如果目标是“计算机识别并结构化给 AI”，建议按下面路线：

1. 重新采集多光照照片，至少 8 个方向斜光。
2. 对每张照片做正射/裁切/去阴影/归一化。
3. 生成多种增强视图：平光、侧光、边缘、法线/曲率、二值化候选。
4. 人工圈出文字区域和列/行；必要时标注少量字框/笔画 mask。
5. 用碑刻二值化模型或微调 U-Net/SegFormer 把字符从石材中分离。
6. 字符检测/切字后，用 PaddleOCR/古汉字模型/视觉大模型生成候选。
7. 用古汉语、地名、人名、纪年、上下文和人工校读做纠错。
8. 输出结构化：原图坐标、字框、候选字、置信度、人工确认字、缺字标记、释文、断句、译文、地理信息。

## 现场重拍建议

最低成本版本：

- 手机或相机固定在三脚架上，镜头尽量正对碑面。
- 关闭自动美颜和 HDR 的不可控处理；能拍 RAW 就拍 RAW。
- 用手电/LED 从 8 个方向斜照，光线与碑面夹角约 10-30 度。
- 每个方向拍一张，再拍一张均匀漫射光。
- 不要让树影落在碑面；如果在室外，阴天或遮光后补光更好。
- 让经纬度、时间、编号写进文件名或旁边白板，不要压到碑面图像里。

进阶版本：

- 做 RTI：固定相机和碑，记录光源位置，使用 RTIBuilder/relight/OpenLIME 生成可交互重光照。
- 做摄影测量：环绕拍多视角，生成点云/网格/正射影像，再计算法线和曲率。
- 对很重要的碑，考虑结构光或激光扫描，保存毫米级三维数据。

## 单张照片的处理流程

当前 `tools/stele_enhance.py` 已实现：

- 红色经纬度叠字检测与修补。
- 灰度化与大尺度背景光照校正。
- CLAHE 局部对比增强。
- 多尺度 unsharp 保留细节。
- Gabor 多方向滤波突出竖/横/斜笔画。
- black-hat/top-hat 形态学提取细笔画能量。
- 两个方向的伪侧光图。
- 自适应阈值二值化候选。
- 输出总览图方便比较。

这套东西适合做初筛和人工读字辅助，但它的二值图不能当作真实字形。

```csv

```

## 最小可行实验

1. 从第二张或第三张图手动裁一个 800-1500 像素宽的文字密集区域。
2. 跑 `tools/stele_enhance.py`。
3. 把 `07_directional_gabor_strokes.png`、`08_morphological_stroke_energy.png`、`09_relief_light_ne.png` 给人工读碑者看，记录哪些字真的变清楚。
4. 如果有效，再批量处理所有照片；如果无效，优先回到现场重拍多光照。
5. 用 PaddleOCR 做一次候选识别，但只当候选，不当定稿。

脚本支持直接传入原图坐标裁剪：

```powershell
python tools\stele_enhance.py inputs\image2_b842e.png -o outputs\sample_b842e_crop --roi 850 100 3000 1900
```

## 下一步工程路线

- 第 1 周：重拍 5-10 块碑，每块 8-16 个方向光照，建立命名规范。
- 第 2 周：整理正射裁切、增强视图和人工可读性评分。
- 第 3 周：选 100-300 个小图块做字符/笔画 mask 标注。
- 第 4-5 周：微调一个碑刻二值化模型，和传统增强作对比。
- 第 6 周：接 PaddleOCR/古汉字识别候选和人工校读界面。
- 后续：把释文、坐标、图像区域、置信度、缺字候选组织成 JSON/TEI，喂给 RAG 或知识库。
