# 项目合并记录

日期：2026-04-28

这次把 `relics-align2` 合并进 `RTI-Learning`，主要是因为两个项目其实在做同一件事。

`RTI-Learning` 原来偏基础学习：RTI、光度立体、采集工具、论文、硬件方案。
`relics-align2` 偏实际问题：几张碑刻照片看不清，后期增强没用，于是转向重新采集、数字拓片、汉画像石结构化和 YOLO。

合并以后，我把 `RTI-Learning` 当主仓库，`relics-align2` 作为一个案例放进去。

## 共同点

两个项目都围绕石质文物表面信息：

- 碑刻文字
- 汉画像石图像
- 浅浮雕
- 风化、磨损、划痕
- 低对比线条
- 榜题和题记

真正的问题不是“图片不够锐”，而是采集阶段没有把足够的表面信息留下来。所以后面的技术重点都落到多光照、RTI、光度立体、三维模型、法线图、曲率图和数字拓片上。

## 为什么没有上传原始照片

原始碑刻照片和处理结果没有放进 GitHub，原因有三个：

- 原图带经纬度水印。
- 文件体积比较大。
- 处理图可以用脚本重新生成，没有必要都放进 Git。

如果以后要公开数据，应该单独做一版脱敏数据集。

## 后面怎么用

我的想法是：

1. 主线文档负责打基础。
2. `case-studies` 负责记录实际案例。
3. 每个案例都写清楚：问题是什么，试了什么，哪里失败，下一步怎么做。
4. 等采集和标注稳定以后，再考虑训练模型或做数据库。
