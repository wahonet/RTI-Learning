# 教程四：从RTI到微痕复现和汉画像石

## 1. 微痕成像和RTI的关系

公开微痕成像资料中反复出现这些关键词：
- 不同入射角度光线。
- 光源矩阵。
- 微痕增强。
- 微米级刻痕。
- 增强图、灰度图、线图、电子拓片。
- 全信息模型、CHIM/IIML、AI识读。

这些描述与RTI高度相关，但不能简单等同。更准确地说：微痕成像是面向文博应用的工程化系统，RTI/光度立体是其中可公开复现的核心技术基础之一。

## 2. 复现策略

不复制未公开商业算法，而是用公开方法复现同类能力：

| 微痕成像公开描述 | 可复现技术 |
| --- | --- |
| 不同入射角光线 | H-RTI或固定光源矩阵 |
| 光源矩阵 | LED阵列/半球光源 |
| 微痕增强 | RTI增强渲染、法线增强、局部对比度 |
| 灰度图 | 虚拟光照合成后转单通道 |
| 线图 | 边缘检测、结构张量、人工校正 |
| 电子拓片 | 灰度/法线/高度图转黑白拓片风格 |
| 全信息模型 | JSON/SQLite对象包 |
| AI识读 | 后续基于标注数据训练 |

## 3. 汉画像石为什么特殊

汉画像石不是单纯文字对象，它同时包含：
- 图像叙事。
- 纹样。
- 人物和器物轮廓。
- 线刻和浅浮雕。
- 可能的铭文。
- 历史拓片和原石之间的差异。

所以输出不能只有“增强图”。至少要考虑：
- 原始照片。
- RTI交互文件。
- 法线图。
- 增强灰度图。
- 数字拓片。
- 线图。
- 图像/文字标注。
- 释读和置信度。

## 4. 数字拓片思路

传统拓片把凹凸表面转化成黑白图像，但有接触损伤风险。数字拓片可以从RTI/法线/高度线索生成。

基本流程：

```text
照片序列
  ↓
RTI或光度立体
  ↓
法线图/增强灰度图
  ↓
背景压制和局部增强
  ↓
阈值、反相、去噪
  ↓
数字拓片
```

需要注意：
- 数字拓片是算法输出，不等于真实拓片。
- 参数必须保存。
- 最好保留多个版本，让研究者比较。

## 5. 线图生成思路

汉画像石研究常需要线图。自动线图可以作为辅助，但不能完全替代人工。

初步方法：
- 从增强图做边缘检测。
- 从法线图寻找方向突变。
- 用形态学方法连接断线。
- 用人工界面修正误识别。

风险：
- 石材颗粒会被识别为线条。
- 裂缝可能被误认为图像轮廓。
- 风化边缘可能断裂。

## 6. 全信息模型思路

一个对象包至少应包含：
- 对象档案。
- 原始照片序列。
- 光照方向。
- 处理参数。
- RTI结果。
- 增强图。
- 数字拓片。
- 线图。
- 标注。
- 释读。
- 版本记录。

这样后续AI识读、专家复核、展览展示和数据发布都能基于同一套数据。
