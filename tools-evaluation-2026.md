# 工具选型与评估 2026

本文用于判断每个工具在RTI/PTM、光度立体、3D、标注、发布和数据管理中的位置。评分是面向本项目的主观优先级，不是通用软件排名。

## 评估维度

- 可用性：是否容易安装和上手。
- 研究价值：是否能直接服务汉画像石、碑刻、微痕和图像学。
- 可复现性：是否适合记录参数和批处理。
- 扩展性：是否能接入后续软件系统。
- 风险：依赖、维护状态、版权、学习成本。

## 处理与查看

### RelightLab / Relight CLI

- 优先级：最高。
- 用途：RTI创建、PTM/HSH/RBF、法线图、Web格式、批处理。
- 优势：当前活跃，支持OpenLIME生态。
- 风险：源码较大，Windows/依赖需要验证。
- 本项目动作：优先下载Release，跑CHI Fish Fossil样例。

### RTIViewer

- 优先级：高。
- 用途：查看PTM/HSH/RTI，学习增强模式。
- 优势：经典，文献和教程多。
- 风险：较老，不适合做长期平台。
- 本项目动作：作为效果对照和术语学习工具。

### RTIBuilder

- 优先级：中。
- 用途：传统H-RTI处理流程参考。
- 优势：和CHI教程高度对应。
- 风险：老旧，现代系统兼容性差。
- 本项目动作：只作为传统流程和格式参考。

### OpenLIME

- 优先级：最高。
- 用途：Web端RTI、多光谱、BRDF、神经重光照、标注展示。
- 优势：现代WebGL、多层、多通道、可扩展。
- 风险：需要前端集成能力。
- 本项目动作：做最小OpenLIME页面，后续接IIML。

## 光度立体与3D

### ImageJ Photometric Stereo Tools

- 优先级：高。
- 用途：从RTI式掠射光图像估计法线和高度。
- 优势：文物保护场景已有使用，Fiji环境易交互。
- 风险：脚本环境和大图性能需要测试。
- 本项目动作：先用硬币/刻痕板测试。

### COLMAP

- 优先级：高。
- 用途：SfM/MVS、相机位姿、点云、整体3D。
- 优势：成熟、开源、命令行强。
- 风险：对纹理、照片质量、GPU和参数敏感。
- 本项目动作：用于整体画像石或样本3D模型。

### Meshroom / AliceVision

- 优先级：高。
- 用途：节点式摄影测量、3D重建、可能的光度立体节点。
- 优势：GUI友好，适合学习流程。
- 风险：对硬件和依赖有要求。
- 本项目动作：与COLMAP二选一做第一轮3D实验。

### CloudCompare

- 优先级：中高。
- 用途：点云裁剪、配准、距离比较、质检。
- 优势：点云分析强。
- 风险：学习曲线较陡。
- 本项目动作：用于比较3D扫描/光度立体/摄影测量结果。

### MeshLab

- 优先级：中。
- 用途：网格清理、简化、纹理和格式转换。
- 优势：传统工具，文博常用。
- 风险：自动化和可追溯较弱。
- 本项目动作：作为3D后处理工具。

## 标注与AI

### AnyLabeling

- 优先级：最高。
- 用途：本地YOLO/SAM/SAM2/SAM3辅助标注。
- 优势：适合个人快速做小数据集。
- 风险：正式协作和审核流程不如CVAT。
- 本项目动作：第一批20-50张汉画像石标注优先用它。

### CVAT

- 优先级：高。
- 用途：正式图像/视频标注、YOLO/COCO导入导出、SAM自动标注。
- 优势：格式全面，协作强。
- 风险：Docker部署和维护成本。
- 本项目动作：当数据集超过100张后评估。

### Label Studio

- 优先级：中高。
- 用途：多类型数据标注、文本和图像混合任务、模型预标注。
- 优势：适合把释读文本和图像标注放一起。
- 风险：CV视觉格式工作流可能不如CVAT直接。
- 本项目动作：评估是否适合IIML审核界面。

### Ultralytics YOLO + SAM

- 优先级：高。
- 用途：检测、分割、自动预标注。
- 优势：生态完整，`auto_annotate`可用YOLO框驱动SAM分割。
- 风险：预训练模型不懂汉画像石，必须人工审核。
- 本项目动作：作为预标注工具，不直接作为最终释读。

## Web发布与互操作

### IIIF

- 优先级：高。
- 用途：高分辨率图像发布、跨系统互操作。
- 优势：文博生态成熟。
- 风险：RTI和多光照数据需要扩展表达。
- 本项目动作：原图、拓片、线图、增强图先走IIIF。

### Annotorious / OpenSeadragon

- 优先级：中高。
- 用途：大图标注、Web Annotation。
- 优势：轻量，适合自研IIML前端。
- 风险：复杂审核和多版本管理需自研。
- 本项目动作：做最小IIML标注器原型。

### Mirador / Universal Viewer

- 优先级：中。
- 用途：IIIF浏览、对比、馆藏展示。
- 优势：成熟Viewer。
- 风险：自定义RTI交互有限。
- 本项目动作：用于对照展示和公开传播，不作为RTI主查看器。

### 3DHOP / Potree

- 优先级：中。
- 用途：3D模型和大型点云Web展示。
- 优势：文化遗产3D展示成熟。
- 风险：与RTI/IIML联动需要额外开发。
- 本项目动作：3D阶段再引入。

## 数据版本管理

### DVC

- 优先级：高。
- 用途：大图、训练集、模型权重和实验输出版本管理。
- 优势：和Git协作，数据不进Git但可追溯。
- 风险：需要配置远端存储。
- 本项目动作：当开始训练YOLO或保存大量RAW时引入。

### Git LFS

- 优先级：中。
- 用途：少量必须跟随仓库的大文件。
- 优势：简单。
- 风险：GitHub配额和带宽限制。
- 本项目动作：不作为主数据管理方案。

## 推荐第一轮工具组合

```text
RelightLab
  + RTIViewer
  + AnyLabeling
  + COLMAP 或 Meshroom
  + JSON-LD/IIML
```

## 推荐第二轮工具组合

```text
Relight CLI
  + OpenLIME
  + ImageJ Photometric Stereo Tools
  + CVAT
  + DVC
```

## 推荐长期工具组合

```text
OpenLIME
  + IIIF
  + Annotorious
  + PostgreSQL/SQLite
  + RDF/Neo4j/Wikibase
  + 3DHOP/Potree
```
