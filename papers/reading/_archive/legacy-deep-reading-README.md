# 论文 HTML 精读覆盖说明

本目录是论文阅读的统一入口，不再保留旧的普通导读层。

- `index.html`：总入口，按卡片进入每篇论文或资料。
- `*_deep.html`：屏幕优先的对照阅读页。左侧用大屏 PDF/来源阅读器，右侧放中文逐页/逐段译读、概念解释和数学补课。
- `paper-data/*.json`：通用联动阅读器使用的逐页导读数据。当前 20 个本地 PDF 均已建立数据文件，共 277 张可高亮定位的导读卡。

## 当前覆盖

- 本目录现有 38 个精读页，覆盖 `docs/05-validation-indexes/research-master-index-2026.csv` 中登记为 `paper` 的条目，并保留 CHI 工具类 PDF 的中文读法。
- 有本地 PDF 的页面会直接在左侧嵌入 PDF，并默认放大到可读尺寸，避免旧版半张 A4 中看不清字的问题。
- 暂时没有 PDF 或直链下载失败的页面，会在左侧放论文页/来源页，并在右侧先做摘要和核心概念精读。
- `07_RTI_as_a_New_Documentation_and_Analysis_Method_in_Epigraphical_Studies_deep.html` 已升级为第一版 PDF.js 精确联动样板：第1页摘要做逐句直译/顺译/词组拆解，第2-16页补了按页句段联动译读卡片。
- `interactive-reader.html` 是通用 PDF 精确联动阅读器，已接入所有本地 PDF 精读页的顶部入口；它会自动抽取当前页句子坐标，并读取 `paper-data/*.json` 生成左文右译的同步高亮导读卡。
- 需要“左侧原文区域与右侧译文句子精确联动”的页面应通过本地 HTTP 打开，例如 `http://127.0.0.1:8765/papers/reading/02-heritage-epigraphy-cases/07-RTI_Epigraphical_Studies/07-RTI_Epigraphical_Studies_guide.html`。直接用 `file://` 打开时，浏览器会拦截 PDF.js 模块。

## 逐段译读规范

后续每篇论文升级时，右侧不再只写摘要，应按以下粒度处理：

- 每个小节按页码拆开，至少覆盖标题、摘要、引言、方法、实验/案例、讨论、结论。
- 每个阅读块包含“原文位置”“关键词”“中文意译”“解释/项目提示”。
- 涉及公式、法线、最小二乘、矩阵、优化、损失函数时，必须链接到数学 HTML 小课。
- 不把整篇论文原文全文搬进 HTML；左侧使用本地 PDF 或来源页承载原文，右侧做逐段中文意译和讲解。
- 有版权不明确的论文只做意译、解释和短关键词定位；公开/自有材料可继续细化。
- 精确一一对应优先使用 PDF.js 从本地 PDF 运行时抽取文字坐标，避免手工估框导致错位。
