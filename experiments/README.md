# 实验复现与记录模板

本目录用于保存RTI/PTM、光度立体、数字拓片、AI标注和Web发布实验记录。每个实验都应能回答：输入是什么、如何处理、输出是什么、哪里成功、哪里失败、下一次如何改。

## 实验命名

建议格式：

```text
YYYY-MM-DD_short-topic_object_method
```

示例：

```text
2026-05-01_coin_h-rti_relightlab
2026-05-03_stone-scratch_fixed-led-16_photometric-stereo
2026-05-05_han-public-yolo_anylabeling
```

## 实验目录结构

```text
experiments/
  2026-05-01_coin_h-rti_relightlab/
    README.md
    inputs.md
    commands.md
    observations.md
    outputs.md
    issues.md
```

大图和输出文件不直接放入Git，只写路径和摘要。

## 第一批必须做的实验

### 实验A：CHI Fish Fossil官方样例

目标：
- 跑通RelightLab或RTIBuilder。
- 输出至少一种RTI格式和法线图。
- 用RTIViewer或OpenLIME查看。

验收：
- 能说明输入照片数、光照方向来源、输出格式。
- 能截图或记录两种增强模式的观察差异。

### 实验B：硬币H-RTI

目标：
- 用手持LED和反光球采集硬币。
- 验证反光球高光估计和低角度光照效果。

验收：
- 得到可交互RTI。
- 能观察硬币文字、磨损和微小划痕。
- 记录失败照片和原因。

### 实验C：刻痕板固定光源矩阵

目标：
- 使用16路或32路LED固定采集。
- 输出法线图、虚拟掠射光图和高度图初稿。

验收：
- 每张照片可对应灯位编号。
- `lights.csv`完整。
- 能比较不同灯位数量对结果的影响。

### 实验D：汉画像石公开图YOLO/SAM标注

目标：
- 选20-50张公开汉画像石图片。
- 标注少量类别。
- 生成YOLO bbox和SAM polygon。

建议类别：
- 伏羲女娲。
- 乐舞人物。
- 车马。
- 神兽。
- 建筑构件。
- 铭文/榜题。

验收：
- 至少导出YOLO格式。
- 至少导出一份IIML候选标注。
- 标明所有图片来源和版权状态。

### 实验E：OpenLIME/IIIF发布

目标：
- 把一张增强图或RTI输出发布成可浏览Web页面。
- 测试标注层叠加。

验收：
- 能在浏览器中缩放查看。
- 能显示至少一个标注区域。
- 明确OpenLIME、IIIF、Annotorious各自作用。

## 质量评分表

每个实验建议按1-5分评分：

- 清晰度：目标线条是否清楚。
- 稳定性：照片是否位移、模糊、过曝。
- 可复现性：元数据是否足够别人重做。
- 研究价值：是否真的帮助识读/线图/标注。
- 风险控制：是否避免误读和伪影。

## 常见失败原因

- 相机或对象移动。
- 反光球过曝。
- 光源方向覆盖不均。
- 环境光太强。
- 低角度光不足。
- 石材纹理被误当成刻痕。
- AI分割把裂缝或污渍当成图像轮廓。
- 标注没有绑定具体图像版本。
